# 试飞控制系统 — 端侧小模型设计(动作编码 → 目标点,机载推理)

> 子系统 spec,基于 `2026-07-01-总体架构-design.md`(2026-07-07 重构后)、`2026-07-03-后端B-design.md` §4。研究内容 2("试飞动作小模型训练")在重构后的落点。
>
> **定位**:端侧小模型是 B 内组件 `small_model`,实现 `GoalGenerator` 接口(总体架构 §3.7)。输入离散环境数据(风速/气温等)+ 无人机状态参数 + `ActionCommand`(动作编码+数值),输出**目标点**(ego-planner 所需输入的子集,先导直接作 setpoint)。先导在 B 侧 venv-B 用**规则 stub** 占位打通链路(不训练);阶段2/4 接 ego-planner + 雷达;阶段4 迁机载电脑 ONNX 推理。
>
> **与 [2026-07-05-小模型α-design.md](2026-07-05-小模型α-design.md) 的区别**(关键,勿混):
> - 那份是"**蒸馏小模型 α**":替代 LLM α 做指令→ActionCommand 翻译,在 A 侧,标的来自 (intent, ActionCommand)。
> - 本 spec 是"**端侧动作小模型**":α 下游,做 ActionCommand+状态/环境→目标点,在 B 侧/机载,标的来自 (ActionCommand+环境+状态, 目标点)。
> - 两者是不同组件、不同位置、不同标的,研究内容 2 字面"试飞动作小模型"在本 spec(动作→控制量层);蒸馏 α 是研究内容 1 的远期增强。
>
> **不阻塞先导**:先导用 stub 占位,不训练真模型;阶段4 外场前训练上线。与后端 B/A/前端编码并行。

## 1. 定位与安全边界

### 1.1 为何要端侧小模型(LLM α 已能产 ActionCommand,为何再有小模型)

| 价值 | 说明 |
|---|---|
| **试飞动作专精** | LLM α 产的是"动作意图"(`goto (3,2,1)`),但"在当前风速/载重下,以什么目标点序列逼近、何时切下个动作"是飞行控制层知识。小模型把动作编码 + 实时状态/环境 → 精确目标点。 |
| **可机载部署/离线** | 研究内容 2 明文"为后续机载部署提供算法基础"。LLM 依赖云端,端侧小模型本地推理,断网可飞——阶段四外场演示前提。 |
| **可复现可审计** | 同样 (ActionCommand, 状态, 环境) 应出同样目标点。LLM 不确定,小模型权重固定,可回归测试。 |
| **延迟可控** | LLM α 1~5s/周期;小模型本地推理 50~200ms,外场响应快。 |
| **接 ego-planner 的桥梁** | ego-planner 要目标点 + 点云;小模型产出"目标点子集",把高层动作意图落到可避障的目标层。 |

### 1.2 不做什么(安全边界,对齐总体架构 §1.3)

- **不输出操纵杆/油门/电机控制量**。那是 PX4 内置控制器职责(原始设计稿四级控制器)。小模型**升一层**只产目标点,交给 ego-planner(阶段2/4)/setpoint 直发(先导)。
- **不绕过 ego-planner**(阶段2/4)。小模型目标点仍经 ego-planner 在线避障;规划失败回 `reject` 上行 α 重想(总体架构 §3.6)。先导无 ego-planner,物理安全靠 `safety_constraints` 夹紧 + 假无人机边界自保(B spec §9.1)。
- **不直接与人对话**。α 是非对话翻译器,小模型更不对话;只经 WS `alpha_output` 系统消息推送目标点(总体架构 §3.3)。
- **不自动执行飞行处置**。β 提议经 `propose_to_alpha` 仍需人审核才注入 α(总体架构 §1.3)。

### 1.3 与 α / ego-planner 的协作链

```
β(中枢)→ propose_to_alpha/forward → α(LLM, 动作翻译器)
   → ActionCommand{actions:[{code,value,target?}], safety_constraints}
   → 经总线 A↔B IPC: bus.call(to="small_model", tool="generate_goal", args={action, pose, env})
   → 端侧小模型(B 先导 stub / 阶段4 机载 ONNX)
   → 目标点 [x,y,z] + speed_max
   → 先导:直接 setpoint → 假无人机
   → 阶段2/4:ego-planner(目标点+雷达点云)→ 避障连续轨迹 → setpoint → PX4
```

- `ALPHA_BACKEND`(选 α 后端:llm/蒸馏 small)与本组件无关——本组件恒为 `small_model`,α 后端切换不改小模型接口。
- 先导 stub→阶段4 真模型切换不改总线协议(`generate_goal` tool 不变),只换 B 内 `small_model/` 实现。

## 4. GoalGenerator 统一接口与先导 stub(对齐 B spec §4)

### 4.1 GoalGenerator 接口

```python
# backend-B/small_model/goal_gen.py
from abc import ABC, abstractmethod

class GoalGenerator(ABC):
    """端侧小模型统一接口(总体架构 §3.7 组件契约, B spec §4)。
    输入: ActionCommand(单条动作) + 当前位姿 + 离散环境
    输出: 目标点 [x,y,z] + speed_max(场地系)
    失败抛 GoalGenError, 目标点线程捕获后下发悬停(B spec §8.1)。"""
    @abstractmethod
    def generate(self, action: dict, pose: dict, env: dict, safety: dict) -> dict:
        """返回 {"goal":[x,y,z], "yaw":float, "speed_max":float}。
        action: {code, value?, target?, units?}(总体架构 §2.2 单条)
        pose: {pos:[x,y,z], quat:[w,x,y,z], vel:[...]}
        env: {wind:[...], temperature, humidity, ...}(离散化)
        safety: {speed_max, ceiling, floor, boundary}
        失败抛 GoalGenError(unknown_action_code / out_of_boundary_after_clamp)。"""
        ...

class GoalGenError(Exception): ...
```

```python
# backend-B/small_model/goal_gen.py 路由
import os
from .stub import StubGoalGenerator
from .onnx_model import OnnxGoalGenerator

def make_goal_generator() -> GoalGenerator:
    backend = os.environ.get("SMALL_MODEL_BACKEND", "stub")  # stub|onnx
    if backend == "onnx":
        return OnnxGoalGenerator(model_dir="models/small_model")
    return StubGoalGenerator()  # 先导
```

目标点线程(B spec §4.3)调 `generator.generate(action, pose, env, safety)`,**不感知后端**。切换 `SMALL_MODEL_BACKEND` 不改线程、不改总线、不改 α。

### 4.2 先导 stub 实现(`stub.py`,规则占位)

先导**不训练**,用规则映射动作编码→目标点(打通链路优先):

| `code` | stub 目标点 | 说明 |
|---|---|---|
| `takeoff` `(value)` | `[home.x, home.y, value]` | 起飞到 value 高度,夹紧 `[floor,ceiling]` |
| `goto` `(target)` | `target` 夹紧 boundary | 绝对目标点 |
| `move` `(value)` | 当前位姿 + 当前朝向单位向量 × value,夹紧 boundary | 沿朝向移动 |
| `climb` `(value)` | `[cur.x, cur.y, cur.z + value]` 夹紧 `[floor,ceiling]` | 相对爬升 |
| `descend` `(value)` | `[cur.x, cur.y, cur.z - value]` 夹紧 `[floor,ceiling]` | 相对下降 |
| `yaw` `(value)` | 目标点=当前位姿,yaw=value | 只改航向 |
| `hover` `(value)` | 当前位姿 | 原地悬停 value 秒 |
| `return_home` | `home.position` | 返航 |
| `land` | `[home.x, home.y, floor]` | 降落 |
| 未知 `code` | 抛 `GoalGenError("unknown_action_code")` → 上行 `reject` | 回告 α 重想 |

```python
# 伪代码
class StubGoalGenerator(GoalGenerator):
    def generate(self, action, pose, env, safety):
        code = action["code"]
        cur = pose["pos"]
        b = safety["boundary"]  # [[min_x,min_y,min_z],[max_x,max_y,max_z]]
        if code == "goto":
            goal = clamp(action["target"], b)
        elif code == "takeoff":
            goal = clamp([home[0], home[1], action["value"]], b, z_range=[safety["floor"], safety["ceiling"]])
        elif code == "climb":
            goal = clamp([cur[0], cur[1], cur[2] + action["value"]], b, z_range=[safety["floor"], safety["ceiling"]])
        # ... 其余编码
        else:
            raise GoalGenError(f"unknown_action_code:{code}")
        if out_of_boundary_after_clamp(goal, b):
            raise GoalGenError("out_of_boundary_after_clamp")
        return {"goal": goal, "yaw": action.get("yaw", 0.0), "speed_max": safety["speed_max"]}
```

- 无 ego-planner:stub 产出的目标点**直接作 setpoint**下发假无人机(B spec §4.3)。
- 物理安全:夹紧 `safety_constraints.boundary` + `[floor,ceiling]` + `speed_max`;假无人机边界自保(§9.1)做最后一道网。**无碰撞检测**(先导不验避障,决策 4)。

### 4.3 small_model 组件执行流程(对齐 B spec §4.3)

```
收到 A 的 call(action, ActionCommand)
  ├─ 取首条/下一条未执行 action
  ├─ generator.generate(action, pose, env, safety) → 目标点
  │   ├ GoalGenError(unknown_action_code / out_of_boundary) → 产 event:reject 上行 A,回 result:rejected
  │   └ 成功 → 目标点入"当前目标"缓存,目标点线程下发 setpoint
  └─ result:ok

目标点线程 tick(20Hz):
  ├ 若有当前目标点: 按 speed_max 限速下发 setpoint(目标位置+yaw)
  │   到达阈值内 → 切下一条 action → status(currentAction/totalActions)
  │   全部完成 → status(completed)
  └ 若无目标 / abort / hover: 下发"当前位姿悬停"setpoint
```

### 4.4 阶段2/4 ego-planner 接入

> 阶段2/4:小模型目标点不再直接作 setpoint,改喂 ego-planner(+雷达点云)出避障连续轨迹。

- `ego_planner_bridge/`:把 `generator.generate()` 产出 + `lidar` 的 `pointcloud` event 喂 ego-planner ROS 节点。
- ego-planner 规划失败(目标被障碍包围)→ 上行 `event:reject`(`reason:"ego_planner_unreachable"`)→ α 重想。
- 详见 [PX4 阶段2 spec](PX4-阶段2-design.md)(G4,远期)。
- 小模型接口 `GoalGenerator` 不变,只换下游(直发 setpoint → 经 ego-planner)。

## 5. 数据集 schema(阶段4 训练用,先导 stub 不涉及)

### 5.1 标的来源

小模型训练标的来自**人审通过的 ActionCommand 执行记录 + 小模型/规则实际产出的目标点 + 飞行结果**。从 `conversations` 表抽(A spec §5.5 自动写):

```sql
-- 训练标的: α 产出的 ActionCommand(人审通过) + 当时状态/环境 + 应产目标点
-- "应产目标点"来源: 先导由 stub/规则产(作弱标的); 阶段3+ 由人/β 标注或强化学习奖励回标
SELECT c_intent.content AS action_command_json,       -- α 产出的 ActionCommand
       c_intent.metadata AS meta,                      -- 含 approved/path/action_code
       c_intent.session_id,
       c_intent.created_at,
       env.data AS env_json,                            -- 当时环境(离散化)
       pose_at(c_intent.session_id, c_intent.created_at) AS pose_json,  -- 当时位姿
       actual_goal_at(c_intent.session_id, c_intent.created_at) AS goal -- 实际下发目标点
FROM conversations c_intent
JOIN environments env ON env.id = session_env(c_intent.session_id)
WHERE c_intent.agent = 'alpha' AND c_intent.role = 'tool_call'
  AND c_intent.metadata->'approved' = 'true'
ORDER BY c_intent.created_at;
```

> **标的来源分层**(先导→阶段4):
> - **先导(stub)**:不训练,无标的。
> - **阶段3(弱标的)**:用 stub/规则产出的目标点 + 飞行成功记录作弱标的(行为克隆 stub 行为,小模型学到"在 X 状态下,动作 Y 该去 Z 目标点")。
> - **阶段4(强标的)**:加人/β 标注 + 强化学习奖励(飞行平稳、到点准确、无 reject)回标,提升质量。

### 5.2 数据集样本格式(JSONL)

```json
{"action":{"code":"goto","target":[3.0,2.0,1.0]},"pose":{"pos":[0.5,0.5,1.0],"quat":[1,0,0,0],"vel":[0,0,0]},"env":{"wind":[0.2,0,0.1],"temperature":25,"humidity":45},"safety":{"speed_max":1.5,"ceiling":2.5,"floor":0.3,"boundary":[[0,0,0],[5,4,3]]},"goal":[3.0,2.0,1.0],"session_id":"20260630143012","approved":true}
```

字段:
- `action`:单条 ActionCommand(总体架构 §2.2)
- `pose`/`env`/`safety`:当时状态/环境/安全约束(小模型输入)
- `goal`:应产目标点(小模型应学的输出)
- `session_id`/`approved`:溯源可审计

### 5.3 数据量目标

| 阶段 | 数据量 | 来源 |
|---|---|---|
| 先导(阶段1~2) | 0(stub) | 不训练 |
| 弱标的(阶段3) | 500~2000 条 | stub 执行 + 飞行成功记录 |
| 强标的(阶段4前) | 2000~10000 条 | 人/β 标注 + RL 奖励回标 + 数据增强 |

### 5.4 数据增强(`augment.py`)

- **状态微扰**:位姿/速度 ±5%(保语义)
- **环境微扰**:风速/气温 ±10%(教模型对不同环境鲁棒)
- **动作同义**:LLM α 离线把同义动作改写(如 `climb 0.5` ↔ `takeoff 当前高度+0.5`)
- **边界场景**:故意生成近 boundary 的目标点,教模型夹紧

### 5.5 训练数据快照隔离(对齐蒸馏 α spec §5.1a,G6)

A 侧运行时用 aiosqlite 异步写;训练脚本(venv-A 同步 SQLAlchemy)不直读热库,先 `sqlite3 .backup` 快照到 `data/small_model_dataset/snapshot_<ts>.db`,只读连快照抽标的。

## 6. 模型与训练(阶段4,先导不涉及)

### 6.1 模型选型(数 MB,机载可行)

甲方要求"数 M 大小神经网络"。选型:

| 方案 | 参数量 | ONNX 大小 | 适用 |
|---|---|---|---|
| **MLP(数值输入)** | 0.1~1M | <5 MB | 输入纯数值(动作编码 one-hot + 状态/环境数值),输出目标点 3 维。**首选**(动作编码本就是离散,无需文本) |
| 小 encoder-decoder(T5-tiny 级) | 10~30M | 10~30 MB | 若输入含文本(动作 comment)。**次选**(甲方"数 M"约束下偏大) |

- **首选 MLP**:输入=[动作编码 one-hot(9 类)+ value + target(3)+ pose(7)+ env(N)+ safety(6)],输出=[goal(3)+ yaw + speed_max]。纯数值,数 MB,机载推理 <50ms。
- 训练用 MSE/Huber loss(目标点回归)+ 物理夹紧约束(夹紧后越界惩罚)。

### 6.2 训练目标

```
L = L_goal + λ_clamp * L_clamp + λ_phys * L_phys

L_goal   : 目标点 MSE/Huber(回归)
L_clamp  : 输出经 safety 夹紧后与未夹紧的偏差(鼓励直接产合法目标点,不靠夹紧兜底)
L_phys   : 输出目标点经 ego-planner(阶段2/4 模拟)或 stub 物理校验失败 → 惩罚
```

- 阶段3 弱标的只用 `L_goal`;阶段4 强标的加 `L_clamp` + `L_phys`(RL)。

### 6.3 评测(`eval.py`)

| 指标 | 定义 | 通过阈值(阶段4) |
|---|---|---|
| **目标点误差** | 输出 goal vs 标的 goal 欧氏距离 | < 0.2m(p95) |
| **物理夹紧通过率** | 输出经 safety 夹紧后合法 | ≥ 99% |
| **ego-planner 规划通过率**(阶段2/4) | 输出目标点经 ego-planner 规划成功 | ≥ 90% |
| **延迟** | ONNX 推理 p95 | < 200ms |
| **可复现** | 同输入同输出(权重固定) | 100% |

### 6.4 ONNX 导出与推理

```python
# small-model/export/to_onnx.py
def export_onnx(model, out_dir):
    model.eval()
    dummy = torch.zeros(1, model.input_dim)
    torch.onnx.export(model, dummy, f"{out_dir}/model.onnx",
                      input_names=["features"], output_names=["goal"],
                      dynamic_axes={"features":{0:"batch"}, "goal":{0:"batch"}},
                      opset=14)
```

```python
# backend-B/small_model/onnx_model.py(阶段4)
class OnnxGoalGenerator(GoalGenerator):
    def __init__(self, model_dir="models/small_model"):
        self.sess = ort.InferenceSession(f"{model_dir}/model.onnx")
    def generate(self, action, pose, env, safety):
        features = self._featurize(action, pose, env, safety)  # one-hot + 数值拼接
        out = self.sess.run(None, {"features": features})[0]
        goal = out[0][:3].tolist()
        if out_of_boundary(goal, safety["boundary"]):
            raise GoalGenError("out_of_boundary_after_clamp")
        return {"goal": goal, "yaw": float(out[0][3]), "speed_max": float(out[0][4])}
```

- 推理用 `threading`(B spec §3 单进程多线程,不引入 asyncio);或先导 stub 同步调用。
- 失败抛 `GoalGenError`,目标点线程(B spec §8.1)捕获后下发悬停。

## 7. 安全校验链(小模型输出绝不直飞无兜底)

```
小模型 generate(action, pose, env, safety) → 目标点 [x,y,z]
   │
   ├─ safety 夹紧(boundary / [floor,ceiling] / speed_max)— stub 与 onnx 都做
   │   夹紧后仍越界 → GoalGenError → reject → 悬停
   │
   ├─ 先导:直接 setpoint → 假无人机(假无人机边界自保兜底, B spec §9.1)
   │
   └─ 阶段2/4:ego-planner(目标点+点云)→ 避障连续轨迹
       规划失败 → reject → α 重想 → 悬停
```

**多层校验**:小模型夹紧(第1层)+ ego-planner 在线避障(阶段2/4,第2层)+ 假无人机/PX4 边界自保(第3层)。**任何一层失败都转悬停**(总体架构 §3.6)。

## 12. 风险与缓解

| 风险 | 缓解 |
|---|---|
| 先导 stub 太简单,链路打通但目标点不"智能" | 先导目标只打通链路,不要求智能;阶段4 真模型替换 |
| 训练数据"应产目标点"无强标的(先导 stub 产的) | 阶段3 用弱标的(行为克隆 stub + 成功记录);阶段4 加人/β 标注 + RL |
| MLP 对复合动作序列建模弱 | 先导 stub 逐条产目标点(B spec §4.3);MLP 也逐条,序列由目标点线程管 |
| 动作编码表后续扩展,小模型输入维度变 | one-hot 扩展(加列),旧模型遇未知编码兜底 `reject`(§4.2);`schema_version` 不动 |
| 机载 ONNX 推理环境与训练不一致 | 导出后用同 val 集在机载(或 venv-B 模拟)验证;onnxruntime 跨平台 |
| 与 stub 切换时行为不一致 | β/α 不感知后端;切换前用同 val 集对比两后端目标点误差 |
