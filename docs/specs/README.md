## 阅读顺序建议

1. **`docs/设计目标.md`** — 研究目标与背景（非 spec，仅作需求背景）。
2. **`docs/开发规划.md`** — 阶段A~M 执行计划与排期（落地的开发路径）。
3. **总体/2026-07-01-总体架构-design.md** — 契约源头，先建立全局视图。
4. **总体/A-B-接口冻结** + **开放式接口规范** — 跨进程契约与接入手册。
5. 按职责选读 **后端A/** 或 **后端B/** 或 **前端/** 详细 spec。
6. **测试与质量-路线图** — 验收门槛与进展约定。
