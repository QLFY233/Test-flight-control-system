### 1.2 验收门槛(先导)

- A 单元 + B 纯函数单元测试全绿。
- `S0`~`S7`(后端 B spec §11)逐阶段通过才进下一阶段。
- B 侧 small_model stub 安全舍入单测:`ActionCommand.safety_constraints` 的 `boundary`/`ceiling`/`floor`/`speed_max` 夹紧充分(夹紧后目标点必在 [floor,ceiling] 高度内且不越 boundary);未知动作编码 → 产 reject 单测覆盖。
- `ActionCommand` schema 单测:动作编码全部 9 类合法、`schema_version=2`、缺字段/类型错被 schema 校验拦截。
- msgpack 跨 venv-A(pip 最新)/venv-B(0.6.2)`use_bin_type=True` 帧互解(S0 验证项)。
- 前端不强制自动化;先导以手动验收 + 浏览器 DevTools 为主(开发计划各 P 阶段验收清单),Playwright 后补。

### 1.3 CI(可选,先导不强制定)

- 仓库可选 `.github/workflows/test.yml`:在 Ubuntu 20.04 容器内跑 A 单元(py3.10)+ B 纯函数(py3.8,不依赖 ROS),不计 ROS/gazebo。
- 先导阶段允许手动跑;阶段三联调后引入 CI。
- 测试审查 skill(原始设计稿"自动测试审查"):用 todo/test-driven-development 型流程,不自动拒绝未覆盖 PR,先导仅提示。

### 1.4 测试目录约定

```

backend-B/tests/        # pytest,纯函数子集(goal_gen/action_codes/stub/monitor thresholds/frames);rospy 相关用 rostest 在联调跑
```
