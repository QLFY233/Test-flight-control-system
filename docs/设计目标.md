# 龙虾模式：民航智能辅助系统构建及无人机验证技术必要性分析

当前民航驾驶舱系统相对封闭，飞行员需查阅大量手册、执行繁琐操作并实时监控数据，认知负荷大。试飞阶段尤为突出，任务执行与数据分析高度依赖人工，效率与准确性受限。大模型技术崛起，尤其以“龙虾”模式（即一个中枢大模型动态调度多个专用小模型/工具）为代表的智能体架构，展现出强大的语义理解、任务编排与工具调用能力。该模式能灵活整合知识问答、任务自动化和定制化数据处理，且支持开放式扩展。构建此类机载智能辅助系统，可有效降低飞行员工作负担、提升试飞安全与效率，并为下一代智能航电系统探索可行技术路径。

# 研究目标

本项目旨在构建一个面向民航机载应用的智能辅助最小系统，形成开放式接口规范，并以无人机为验证平台进行全流程演示，验证系统协同能力以及向民航迁移的可行性。核心目标是证明“中枢大模型调度多组件”的混合架构，能在飞行环境中可靠实现知识问答、任务自动执行与数据监控处理等民航典型需求。

# 研究内容

1.智能调度与问答引擎：基于大模型搭建智能体，实现语音/文字交互式问答、任务解析与调度等。

2.试飞动作小模型训练：构建飞行动作数据集，训练轻量化小模型，能够根据试飞任务单中的目标参数，输出操纵杆、油门等控制指令序列，完成规定机动。

3.数据异常监控程序：开发数据监控程序，接入飞行数据流，依据预设阈值与趋势分析逻辑检测异常，并触发大模型调度，生成告警信息与处置建议。

4.指令驱动的数据处理：利用大模型的函数调用能力，将用户的自然语言指令解析为对特定数据处理程序（如傅里叶变换分析）的调用，并返回处理结果。

5.开放式架构设计：定义统一的组件注册、发现与通信协议，使中枢大模型能调度各类小模型与程序，实现功能扩展。

# 研究路径

研究分为四个阶段。阶段一：需求与架构设计。深入分析试飞与驾驶舱场景，设计基于“龙虾”模式的系统顶层架构，明确大模型、小模型、程序间的交互协议与开放式接口标准。阶段二：核心组件开发。调用云端大模型API实现调度中枢与问答模块；采集试飞数据训练动作小模型；编写数据监控和处理程序；集成语音识别与合成接口。阶段三：集成联调与地面仿真：在无人机半实物仿真平台中连接云端，打通“任务下发→大模型解析→小模型执行动作→数据监控告警→语音调用处理”的全链路，消除集成问题。阶段四，外场无人机演示与评估：开展实际飞行演示，测试不同工况下链路延迟、动作执行准确率、异常检测响应时间及扩展能力，全面评估系统可行性。

# 预期成果/产出

1\. 无人机演示验证系统：一套运行在无人机上的智能辅助原型，完整体现民航场景下四项核心功能的闭环协同。

2.专用小模型：一个可生成基本试飞操纵指令序列的小模型，经无人机验证后可为后续机载部署提供算法基础。

3.功能程序与中间件：包括实时异常监控、指令驱动数据处理程序，以及“云端-本地”可靠调度通信组件。

4.开放式接口规范：一份定义组件接入标准的文档，指导后续新模型与程序的快速集成。
