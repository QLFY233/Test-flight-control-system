## P0 — 项目骨架 + 基础设施 (9 文件)

**目标：** `index.html` 在浏览器打开不报错，路由能切页面，WebSocket 能连上后端。

| 文件 | 内容 |
|---|---|
| `index.html` | SPA 入口：`<div id="app">`、`<script type="module" src="js/app.js">`、CSS 引用 |
| `css/reset.css` | `* { margin:0; padding:0; box-sizing:border-box }` + 全局深色背景 `#000` |
| `css/variables.css` | 全部 CSS 自定义属性 (见 §15 详细设计) |
| `js/state.js` | Store 类：`subscribe/set/batch/get`，初始化空状态树 |
| `js/router.js` | Hash 路由：监听 `hashchange` → `mount/unmount` 页面控制器 |
| `js/config.js` | 配置加载器：`loadConfig()` → fetch `config-default.json`，与 localStorage 保存的用户配置 deep merge，挂载到 `window.__config` |
| `js/ws.js` | WebSocket 封装：`connect/disconnect/send/on`，指数退避重连 |
| `js/app.js` | 入口：初始化 Config/Store/WS/Router，挂载根布局，默认导航到 `/` |
| `config-default.json` | 默认配置（见下方格式约定） |

**验收标准：**
- [ ] `index.html` 打开后页面为纯黑背景，顶部显示"连接中..."
- [ ] 修改 URL hash → 控制台打印页面切换日志
- [ ] WebSocket 尝试连接（即使后端未运行，应显示"连接断开"）

## Config 文件格式约定

`config-default.json` 作为默认配置基准，用户自定义配置通过 `localStorage` 覆盖，运行时由 `js/config.js` 做 deep merge。

### 可扩展格式

```
_format: “flight-control-config”   — 固定标识
_version: <number>                 — 版本号，向后兼容
_comment: “<string>”              — 说明文字
顶层按类别分组: backend / llm / speech / display / ...
```

- `_` 开头的 key 为元数据，deep merge 时跳过，不覆盖。
- 分组嵌套不限层级，新增类别直接在顶层加 key 即可。
- 每个分组可写 `_comment` 做说明，不影响逻辑。

### config-default.json

> **注意:** 第三方 API key（DeepSeek、讯飞）一律由**后端 A** 持有和调用（总体架构 §1.3 安全边界、§3.9/§3.10）。前端 config **不存放任何第三方 API key**，仅存前端可配置的连接端点与界面偏好。

```json
{
  "_format": "flight-control-config",
  "_version": 1,
  "_comment": "所有字段按类别分组，可自由扩展；第三方 API key 由后端 A 管理，不在此处",

  "backend": {
    "_comment": "后端 A 连接配置",
    "base_url": "http://localhost:8000",
    "ws_endpoint": "/ws",
    "sse_alpha": "/chat/alpha",
    "sse_beta": "/chat/beta"
  },

  "display": {
    "_comment": "界面偏好",
    "theme": "dark",
    "language": "zh-CN"
  }
}
```

### 加载流程

1. `js/config.js` 启动时 `fetch('config-default.json')` 获取默认值。
2. 尝试从 `localStorage` 读取用户保存的配置 (`key: flight-control-config`)。
3. **deep merge**：用户配置逐字段覆盖默认值，`_` 开头的元数据字段不覆盖。
4. 结果挂载到 `window.__config`，全局可读取。
5. SettingsPage 修改后保存到 `localStorage`，页面刷新后重新 merge。

### 扩展方式

- 新增配置类别：在 `config-default.json` 顶层加新 key，SettingsPage 加对应 Tab 即可。
- 新增字段：在对应分组内加 key，SettingsPage 加表单行。
- 敏感字段（如未来需要前端持有的凭据）：`<input type="password">`。但第三方 API key 一律不存前端，由后端 A 管理。

## 构建说明

本前端为**纯静态 HTML/JS/CSS**，由后端 A (FastAPI) 通过 `StaticFiles` 托管。仓库结构（以总体架构 §4.4 为准）：

```
flight-control-system/
├── frontend/                    # 本目录所有文件放在此处
├── backend-A/                   # Agent 中枢代码
├── backend-B/                   # ROS 桥 + 求解器代码
...
```

开发者只需：

1. 将所有文件放入 `frontend/` 目录
2. 后端 A 注册：`app.mount("/", StaticFiles(directory="frontend", html=True), name="frontend")`
3. 浏览器访问 `http://localhost:8000` 即可

开发期间可用 VSCode Live Server 或 Python `http.server` 独立调试。
