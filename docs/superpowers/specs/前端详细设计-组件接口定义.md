## 2. 全局状态管理 (`state.js`)

单一全局 Store，发布订阅模式。所有组件通过 `store.subscribe(path, callback)` 订阅变化，通过 `store.set(path, value)` / `store.update(path, partial)` 修改。

### 2.1 状态结构

```typescript

  field: {

  environment: {

    presets: Record<string, Partial<EnvData>>;

  // —— UI 状态 ——
  ui: {
    activePage: 'overview' | 'beta' | 'alpha' | 'history' | 'settings' | 'dashboard';
    isMobile: boolean;
    keyboardInputFocused: boolean;          // Chat 输入框是否聚焦(用于阻止全局快捷键触发)

    viewLocked: boolean;                    // 用户是否手动锁定了排列

    // —— 悬浮球 ——
    floatingBall: {
      visible: boolean;                    // 默认 true
      expanded: boolean;                   // 扇叶菜单是否展开
      shortcuts: ShortcutItem[];           // 当前快捷键列表
      editing: boolean;                    // 是否在编辑模式
    };
  };
}
```

```typescript
interface Store {

  // 读取整个 state (谨慎，只应在初始化或调试时用)
```

### 3.2a PCM 音频采集(AudioWorklet,A15)

> 前端录音必须出 **PCM 16k/16bit/单声道**(对齐 A spec §4.4)。MediaRecorder 默认 webm/opus,不能直接出 PCM,故用 AudioContext + AudioWorklet。

实现:
- `navigator.mediaDevices.getUserMedia({audio:{sampleRate:16000, channelCount:1, echoCancellation:true}})` 取麦克风流(需 HTTPS 或 localhost)。
- AudioContext(采样率尽量 16000;否则在 Worklet 内重采样),`audioWorklet.addModule('js/audio/pcm-worklet.js')`。
- AudioWorkletNode `process(inputs)` 把 32-bit float 下采样到 16k/16-bit PCM,每 40ms(~1280 字节)调 `port.postMessage` 传出。
- 主线程把 PCM 帧 base64 → WS `voice_frame{type,data,seq,ts}` 上行(A spec §4.4);首帧随带 `auto_send` 设置项(A16)。
- 录音边界:按下麦克风按钮 → `audioContext.resume()`(autoplay policy,需 user gesture)→ 启动 WorkletNode;松开 → `node.disconnect()`,末帧由后端按 seq 判定。

性能:Worklet 在音频线程跑,不阻塞主线程;40ms 帧低延迟。降级:Worklet 不可用时回退 ScriptProcessorNode(已 deprecated,仅兜底)。

## 4. SSE 消息契约 (`sse.js`)

### 4.1 SSE 模块接口

```typescript
interface SseManager {
  // 连接某个 SSE endpoint
  connect(endpoint: '/api/chat/beta', callbacks: {
    onMessage: (chunk: string) => void;   // 文字流片段
    onToolCall: (toolCall: ToolCallInfo) => void;
    onComplete: () => void;
    onError: (err: Error) => void;
  }): void;

  // 断开
  disconnect(endpoint: '/api/chat/beta'): void;

  // 发送用户消息 (REST POST + SSE 接收流式回复)
  sendMessage(endpoint: '/api/chat/beta', text: string): Promise<void>;
}
```

> **传输方式(A9)**:`/api/chat/beta` 是**一次 `POST`,Response 为 `text/event-stream`**(server-sent events 流式回复);不存在"先建立 SSE 长连接再发消息"路径。`SseManager.sendMessage` = POST + 订阅响应流;`connect` 内部即一次 POST 的 SSE 响应,流结束即断开,下次发送是新 POST。

### 6.7 FlightPlanCard — 飞行计划卡片

```typescript
// │  ┌──────────────────────────────┐  │
// │  │ ① 发送 "上升 0.5 米" (Chat) [✕]│  │
// │  │ ② 等待 500ms         [✕]    │  │
// │  │ ③ 发送 "悬停" (Chat)    [✕]    │  │
// │  │ [ + 添加操作 ]               │  │
// │  └──────────────────────────────┘  │
```

## 12. 异常与边界处理

| 场景 | 处理方式 | 涉及模块 |
|---|---|---|
| WebSocket 连接失败 | 指数退避 (1-2-4-8-16-30s cap)，更新 store.connection | ws.js |
| WebSocket 连接中收到消息 | 丢弃（状态不一致） | ws.js |
| WebSocket 断连后重连成功 | 发送 `sync` 消息请求全量状态 | ws.js |
| SSE 断流 | ChatPanel 显示"连接中断，正在重连"，自动重连 | sse.js |
| REST 请求失败 (4xx/5xx) | 静默重试 1 次，仍失败则显示 toast 错误提示 | api.js |
| Three.js WebGL 不支持 | 检测 `WebGLRenderingContext`，显示降级提示"3D 场景不可用" | Scene3D.js |
| 视频帧解码失败 | 跳过该帧，计数错误 | VideoPanel.js |
| 移动端旋转屏幕 | 监听 `orientationchange` / `resize`，重新布局 + ECharts resize | app.js |
| 浏览器标签页隐藏 | 暂停动画循环 (Page Visibility API)，切回时恢复 | Scene3D.js |
| 10Hz 数据积压 | 3D 插值追赶，不逐帧回放；ECharts 丢弃旧点 | 各模块 |
| **键盘**全局快捷键与 Chat 输入冲突 | 仅在 `store.ui.keyboardInputFocused === false` 时触发**键盘**全局快捷键 | app.js |
| 悬浮球**点击**/扇叶点击 | 不受 `keyboardInputFocused` 屏蔽(物理点击独立路径),Chat 输入聚焦时仍可触发(A14) | FloatingBall.js |
| 页面关闭/刷新 | beforeunload 时断开 WebSocket 和 SSE | app.js |
| 小模型/ego-planner reject (ws `reject`) | toast 告警 + ChatPanel 追加"动作 N 被拒绝: {reason}",α 重新翻译或告警人(2026-07-07: 不再有"段") | ChatPanel.js / app.js |
| A↔B 链路断连 (ws `link_status` state=down) | StatusBar 显示 B 链路红色告警 + ConnectionOverlay 提示"飞控失联,无人机已安全悬停";link 加 `small_model`/`ego_planner`(2026-07-07) | StatusBar.js / ConnectionOverlay.js |
| 大模型/语音/小模型 API 失败 (ws `link_status` state=down link=llm/speech/small_model) | StatusBar 对应项告警 + ChatPanel 提示"大模型/小模型不可用,维持悬停" | StatusBar.js / ChatPanel.js |
| α 安全悬停 (status `flightStatus`='hovering') | BottomBar 显示"安全悬停中"(黄色),飞控总有合法期望值 | BottomBar.js |
| 看板配置失败 (ws `dashboard_config` 解析失败/数据不可用) | DashboardPanel 显示"无数据"占位,不影响飞行 | DashboardPanel.js |

### 13.2 人类操控流程(经 β→α 调度)

> 人类只与 β 对话。操控路径:人在 β Chat 打字 → β 调 forward/propose → α 翻译→端侧小模型→目标点→假无人机(2026-07-07)。

```

  ├── 2. β 理解意图 → 调 forward_last_human_message (或 propose_to_alpha → 人确认)
  ├── 3. 指令经总线入 α 队列 → α loop 下周期翻译成 ActionCommand{code:"climb",value:0.5}
  ├── 4. 经总线发 small_model → 产目标点(夹紧 safety_constraints) → setpoint → 假无人机

  └── 5. 位姿更新 → WS pose → 前端刷新 3D/图表
```

### 13.3 β 生成 → 批准 → α 翻译 → α 执行

> 与总体架构 §2.1 翻译链与中枢调度对齐(2026-07-07)：β 产出 FlightPlan(动作意图概要，人审核)→ α 翻译成 ActionCommand → 端侧小模型产目标点。前端**不**把 FlightPlan 直接当 ActionCommand。

```

  ├── 1. 用户输入 "从起点起飞，飞到 (3,2,1) 悬停 2 秒返航"
```

## 16. 全局事件总线

除了 Store 的状态管理，组件间还需要事件总线来处理一次性动作：

```typescript
interface EventBus {
  // 路由 & 通知
  on(event: 'navigate', handler: (page: string) => void): void;
  on(event: 'show-toast', handler: (msg: { text: string; level: 'info'|'warn'|'error' }) => void): void;

  // 飞行计划
  on(event: 'plan-approved', handler: (plan: FlightPlan) => void): void;

  // 安全
  on(event: 'emergency-hover', handler: () => void): void;

  // 悬浮球快捷键
  on(event: 'shortcut-triggered', handler: (shortcut: ShortcutItem) => void): void;
  on(event: 'shortcut-record-start', handler: () => void): void;
  on(event: 'shortcut-record-stop', handler: (actions: RecordedAction[]) => void): void;

  // 视图模式
  on(event: 'view-mode-changed', handler: (mode: 1 | 2 | 3) => void): void;
  on(event: 'view-source-changed', handler: (slotIndex: number, source: string) => void): void;

  emit(event: string, data?: any): void;
  off(event: string, handler: Function): void;
}

// 使用场景:
//   - 'navigate'              → router.navigate()
//   - 'show-toast'            → 显示短暂提示 (REST 失败等)
//   - 'plan-approved'         → BetaPage 通知全局, 触发跳转到 α
//   - 'emergency-hover'       → 任意页面触发悬停 (全局快捷键)
//   - 'shortcut-triggered'    → FloatingBall → AlphaPage 执行操作序列
//   - 'shortcut-record-start' → 开始记录用户操作 (ShortcutEditor)
//   - 'shortcut-record-stop'  → 停止记录, 返回操作序列
//   - 'view-mode-changed'     → ViewModeSelector → 更新 store.ui.viewMode
//   - 'view-source-changed'   → ViewPanel 源切换 → 重新挂载渲染器
```
