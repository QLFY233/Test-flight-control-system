# 试飞控制系统 — 后端 A 设计(Agent 中枢)

> 子系统 spec,基于 `2026-07-01-总体架构-design.md`。后端 A 是 Agent 中枢,运行双 agent(α/β)、托管前端、对接讯飞语音、经 IPC 连后端 B。
> Python 3.10+ venv,FastAPI + Pydantic AI,单进程 async。

## 2. 双 Agent — α/β 初始化与调用

### 2.1 Agent 工厂(多 provider 扩展)

```python
# agents/llm.py
PROVIDERS = {
    "deepseek": {"base_url": "https://api.deepseek.com", "api_key_env": "DEEPSEEK_API_KEY", "model": "deepseek-v4-flash"},
    # 未来加 OpenAI 兼容格式的 provider 在这里加一项
}
def make_agent(instructions, tools=None, output_type=None) -> Agent:
    provider = os.environ.get("LLM_PROVIDER", "deepseek")
    cfg = PROVIDERS[provider]
    model = OpenAIModel(cfg["model"], base_url=cfg["base_url"], api_key=os.environ[cfg["api_key_env"]])
    return Agent(model, instructions=instructions, tools=tools, output_type=output_type)
```

- 切 provider 只改环境变量。
- 只要目标 API 是 OpenAI 兼容格式,加一项配置即可。
- API key 一律环境变量,不入库不入 config 文件。

### 2.2 β Agent

- 启动时建单例,注册 6 个工具(来自 `tools/beta_tools.py`),`output_type=FlightPlan`。
- 按前端请求触发(非 loop),调用 `run_beta(session_id, user_message)`。
- 历史按需带(默认本次 session 全部,β 低频)。
- 产出 FlightPlan(结构化),前端 FlightPlanCard 展示。
- 调用经 `run_agent_with_log`(§5.5)自动写 conversations。

### 2.3 α Agent — 核心

**输出类型:** `AlphaOutput`(结构化,同时承载对话回复和操控意图):

```python
class AlphaOutput(BaseModel):
    reply_to_human: str                       # 对人的回复(展示在 Chat)
    trajectory: TrajectorySpec | None = None  # 翻译的目标点(发给 B),无则 None(纯聊天)
```

**α loop 伪代码:**

```python
async def alpha_loop(app_state, ipc):
    while True:
        await asyncio.sleep(app_state.config.alpha_loop_period)  # 1~5s 可配
        pose = app_state.current_pose
        human_inputs = app_state.drain_human_input_queue()
        # 仲裁 + 翻译
        if human_inputs:
            spec, reply = await translate(alpha_agent, human_inputs, pose)  # 产出 AlphaOutput
        elif app_state.pending_plan:
            spec, reply = await translate_plan(alpha_agent, app_state.pending_plan, pose)
            app_state.current_trajectory_plan = spec; app_state.pending_plan = None
        elif app_state.current_trajectory_plan and app_state.current_trajectory_plan.has_remaining():
            spec = app_state.current_trajectory_plan.next_segment(); reply = None
        else:
            spec = TrajectorySpec.hover(pose); reply = None
        # 下发
        if spec: await ipc.send_trajectory(spec)
        if reply: await ws_push_alpha_output(reply)  # 经 WS alpha_output 推前端
        await repos.save_conversation(app_state.session_id, 'alpha', ...)
```

**大模型非阻塞:** 翻译是 await,但用 task 引用 + 周期检查;未完成则下发悬停或上次意图,不卡 loop。

**α 输入四类:**
1. 人类打字(Chat 文本,经 `/api/chat/alpha` 入队列)
2. 语音输入(前端采 PCM → WS voice_frame → 讯飞 STT → 文字 → 入队列)
3. β 计划转送(人审核通过 → `pending_plan` → loop 翻译)
4. 无人机数据(B 上行 pose/telemetry,loop 每周期读)

**Chat 异步塞队列:** 前端发 `/api/chat/alpha` 只入队列,不立即回复;α 的回复在**下个 loop 周期**的结构化 `AlphaOutput.reply_to_human` 里,经 WS `alpha_output` 消息推送。符合"人类输入放下个 loop 周期"。

### 2.4 β 计划注入 α

前端 `POST /api/plan/approve` → 后端存 `app_state.pending_plan` → α loop 下周期检测 → 用大模型把 FlightPlan(自然语言 summary+航点概要)翻译成 TrajectorySpec(带逐段约束的点序列)→ 分段执行。

## 3. 共享状态 AppState 与 A↔B IPC

### 3.1 AppState(`state.py`)

```python
@dataclass
class AppState:
    config: Config
    session_id: str | None = None
    current_pose: PoseData | None = None
    human_input_queue: list[str] = field(default_factory=list)
    current_trajectory_plan: TrajectoryPlan | None = None
    pending_plan: FlightPlan | None = None
    pending_translation: asyncio.Task | None = None
    last_intent: TrajectorySpec | None = None
    ipc_connected: bool = False
    flight_status: str = "idle"  # idle/hovering/planned/executing/completed/aborted
```

并发保护:`asyncio.Lock`;高频位姿更新用单独锁,避免与低频队列操作互锁。

### 3.2 IPC 协议(`ipc/protocol.py` + `frames.py`)

**消息类型:** A→B = trajectory/abort/hover/ping;B→A = pose/telemetry/status/reject/pong。`SCHEMA_VERSION=1`。

**帧编解码:** `[4字节长度][msgpack 载荷]`,统一 msgpack(两边 venv 都有)。

```python
async def send_frame(writer, msg):
    payload = msgpack.packb(msg)
    writer.write(struct.pack(">I", len(payload)) + payload)
    await writer.drain()

async def recv_frame(reader):
    length = struct.unpack(">I", await reader.readexactly(4))[0]
    return msgpack.unpackb(await reader.readexactly(length), raw=False)
```

### 3.3 IPC Server(`ipc/server.py`)

A 作 server,接受 B 连接:

```python
class IpcServer:
    async def start(self, path="/tmp/flight_control.sock"):
        self.server = await asyncio.start_unix_server(self._handle_client, path=path)
        asyncio.create_task(self.heartbeat_loop())  # 2s ping
    async def _handle_client(self, reader, writer):
        app_state.ipc_connected = True
        try:
            while True:
                msg = await recv_frame(reader)
                await self._dispatch(msg)
        except ConnectionError:
            app_state.ipc_connected = False
            await ws_broadcast({"type":"link_status","link":"A-B","state":"down"})
```

- B 侧恒定时间重连(非指数退避)。
- 心跳:A 每 2s 发 ping,B 回 pong;A 超 5s 无 pong 视为断连。

### 3.4 B→A 消息分发

| 收到 | 处理 |
|---|---|
| `pose` | 更新 `current_pose`(加锁)+ 转发前端 WS + 进遥测缓冲 |
| `telemetry` | 同 pose,进遥测缓冲(每秒批量入库) |
| `status` | 更新 `flight_status` + 转发前端 WS |
| `reject` | 写入 α 上下文(下轮重想)+ 转发前端 WS |
| `pong` | 更新心跳时间戳 |

### 4.2 SSE — Chat 流式(loop 驱动)

```python
@router.post("/api/chat/{agent}")
async def chat(agent, req: ChatRequest):
    if agent == 'alpha':
        app_state.human_input_queue.append(req.message)  # 入队列,不立即回复
        return  # 204,回复在下个 loop 周期经 WS alpha_output 推送
    elif agent == 'beta':
        async def stream():
            async for chunk in run_agent_with_log(beta_agent, req.session_id, 'beta', req.message):
                yield SSE_FORMAT(chunk)
        return StreamingResponse(stream(), media_type=SSE_CONTENT_TYPE)
```

- α:发消息只入队列,回复在 loop 产出的 `AlphaOutput.reply_to_human` 里,经 **WS `alpha_output` 消息**推送(前端已有 WS 连接,复用,不另开 SSE 端点)。
- β:请求驱动流式(Pydantic AI UIAdapter)。

### 4.4 语音处理

**STT(`speech/stt_client.py`):**

```
前端按住语音按钮 → 采 PCM 16k/16bit/mono → 每 40ms base64 一帧 → WS voice_frame 上行
后端 A:
  1. 首个 voice_frame → 生成签名(auth.py) → 连 wss://iat.xf-yun.com/v1?...
  2. 发首帧 JSON(header.status=0, parameter.iat.domain=slm/language=zh_cn/dwa=wpgs, payload.audio.status=0)
  3. 后续 voice_frame → 转发(中间帧 status=1, seq 递增)
  4. 松开/60s 上限 → 发末帧(status=2, audio="")
  5. 收讯飞流式返回 → 解析 payload.result.text(base64→JSON: sn/ws/cw)
  6. 处理动态修正(dwa=wpgs):apd=追加, rpl=替换[rg]范围
  7. 拼接文本 → WS voice_stt_result(is_final=false) 实时回填前端
  8. 末帧返回(status=2) → 最终文本 → voice_stt_result(is_final=true) → 入 human_input_queue
```

**TTS(`speech/tts_client.py`):**

```
α 产出 AlphaOutput.reply_to_human 且前端开启朗读 / 前端发 tts_request
后端 A:
  1. 连 wss://cbm01.cn-huabei-1.xf-yun.com/v1/private/mcd9m97e6
  2. 鉴权方式一:header x-api-key: $XF_API_PASSWORD(免签名)
  3. 流式发文本(>10字可分段,status 0/1/2,seq 递增;parameter.tts.vcn=x6_lingxiaoxuan_flow, audio.encoding=lame, sample_rate=24000)
  4. 收流式 base64 mp3 帧 → WS voice_tts(format=mp3, seq) 下发
  5. 前端 AudioPlayer 解码 mp3 播放
```

**鉴权(`speech/auth.py`):**
- STT 必须签名:`signature_origin = "host: iat.xf-yun.com\ndate: {date}\nGET /v1 HTTP/1.1"`,hmac-sha256(apiSecret),base64,拼 authorization。date 用 RFC1123 UTC,偏差≤300s。
- TTS 用 `x-api-key: APIPassword` 方式一,免签名。

**密钥(`speech/xfyun_config.py`):** `XF_APP_ID`/`XF_API_KEY`/`XF_API_SECRET`/`XF_API_PASSWORD` 从环境变量读,不入库不入 config。

**音频格式契约:**

| 环节 | 格式 |
|---|---|
| 前端采集 | PCM 16k/16bit/单声道 |
| 前端→A 上行 | base64 PCM,每帧 ~1280 字节(40ms) |
| A→讯飞 STT | 原样转发(encoding=raw) |
| 讯飞→A TTS 返回 | base64 mp3(lame,24k) |
| A→前端 TTS 下行 | base64 mp3,format=mp3 |

## 6. 配置加载、生命周期、错误处理

### 6.1 配置加载(`config_loader.py`)

Pydantic 模型校验(`FieldConfig`/`DefaultConstraints`/`Config`),启动时加载 field.yaml + default_constraints.yaml 到内存,存入 AppState。校验失败立即报错退出。Config 含 `alpha_loop_period`(默认 2.0s)、`alpha_history_rounds`(默认 10)。

### 6.2 生命周期(`lifecycle.py`)

**启动 9 步(Uvicorn lifespan):**
1. 加载配置(Pydantic 校验)→ Config 单例
2. 初始化 SQLAlchemy engine + session factory
3. `create_all()` 建表
4. 初始化 AppState(dataclass + Lock)
5. 初始化 α/β Agent(读 prompts,注册 β 工具)
6. 启动 A↔B IPC server(监听 Unix socket)
7. 启动 α loop 后台 asyncio Task
8. 启动心跳(2s ping)
9. 开始接收前端请求

**关停:**
1. 停止接收新请求
2. cancel α loop task(等当前周期结束)
3. 经 IPC 给 B 发 hover(安全悬停)
4. 关闭 IPC(通知 B)
5. 关闭 SQLAlchemy engine
6. 清理资源

### 6.3 错误处理矩阵

| 场景 | 处理 |
|---|---|
| 配置文件缺失/校验失败 | 启动 raise,服务不启动 |
| LLM API 失败 | α 当前轮翻译失败→下发悬停(保安全)+ WS link_status 告警;β 当前回合失败→SSE 返回错误事件 |
| 讯飞 STT WS 断开 | 当前识别中止,前端 voice_stt_result 错误状态;不影响飞行 |
| 讯飞 TTS 失败 | 跳过本次朗读,记日志;不影响飞行 |
| A↔B IPC 断开 | ipc_connected=False;α loop 下发悬停(意图丢弃但不崩);前端 WS link_status state=down |
| B 心跳超时(5s) | 视为断开,同上 |
| 数据库写失败 | 记日志,重试 1 次;遥测丢失不致命,对话丢失告警 |
| WS 客户端断开 | 清理该连接订阅;不影响其他 |
| α loop 异常 | catch + 记日志 + **退避重启**(1~5s 退避,避免疯狂重启);期间 B 收不到意图维持悬停 |

**核心:** 任何失败不让服务崩,降级到安全悬停 + 告警。
