# A↔B 接口冻结(跨进程 IPC 契约)

> 冻结日期:2026-07-05;**2026-07-07 架构重构**:`SCHEMA_VERSION` 升 **1→2**(求解器路径废弃,改端侧小模型;A↔B `call.tool` `trajectory`→`action`;`reject` 语义改为"小模型/ego-planner 无法生成可达目标")。

> 一经冻结,两侧实现必须逐字一致;若需变更,**必须升级 `SCHEMA_VERSION` 并同步两侧**,不静默改。本文件权威性等同于总体架构 §3.2/§3.7。

## 2. 消息字段(统一总线格式,与总体架构 §3.7 逐字一致)

```json
{
  "schema_version": 2,
  "from": "beta",
  "to": "small_model",
  "msg_type": "call",
  "call_id": "uuid",
  "tool": "generate_goal",
  "args": { },
  "payload": { },
  "ts": 1234567890
}
```

| 字段 | 冻结语义 | A↔B 跨进程取值 |
|---|---|---|
| `schema_version` | 协议版本(跨进程一致) | `2`(整数,2026-07-07 升级:求解器路径废弃) |
| `from` | 发送方 | A→B:`"A"` 或 A 内组件名(`"alpha"`/`"beta"`);B→A:`"B"` 或 B 内组件名(`"small_model"`/`"monitor"`/`"ego_planner"`/`"lidar"`)。语义:谁发的,便于 A/B 内分发,不影响跨进程路由 |
| `to` | **接收方组件名**(不是进程名) | 见 §3 路由表 |
| `msg_type` | `call` / `result` / `event` / `error` | A→B 当前只用 `call`;B→A 只用 `event`(`result` 经 IPC 暂不要求,A→B call 当前是 fire-and-forget 下发,见 §4 注) |
| `call_id` | 请求-响应配对 | A 侧生成 UUID;B 侧 `result` 回填同 id(若实现回 result) |
| `tool` | 工具/事件名 | 见 §4(call.tool)与 §5(event.tool)两张独立表 |
| `args` | call 调用参数 | A→B call 携带(`action` call 携 `ActionCommand`+pose+env) |
| `payload` | event 事件数据 | B→A event 携带 |
| `ts` | 时间戳 | 浮点 Unix 秒 |

> **范围**:本节消息格式仅用于 **A↔B 跨进程 IPC**。A→前端 WS 消息(总体架构 §3.3)不用本总线格式,字段名按前端 §3.3 定义(`type`/`schema_version`/`pos`/`quat`/...),A 侧转发时做格式转换。`from`/`to`/`msg_type`/`call_id` 不出现在前端 WS 消息里。

## 3. `to` 路由表(跨进程方向)

`to` 是**接收方组件名**,由对方进程内的总线路由表分发;不是进程名 `"A"`/`"B"`(那是 `from` 的语义)。

### A→B(`to` 由 B 侧 `bus/registry.py` 路由)

| `to` | 接受 `tool` | B 侧处理 |
|---|---|---|
| `"small_model"` | `generate_goal`/`abort`/`hover` | B 内总线调 small_model 组件(后端 B spec §2.2);**先导 stub 占位,阶段4 迁机载真模型** |
| `"ego_planner"`(阶段2/4) | `plan` | B 内总线调 ego_planner 桥(先导不注册);输入目标点+点云→连续轨迹 |
| `"monitor"` | (无,monitor 不接受 call,只产 event) | — |

### B→A(`to` 由 A 侧 `bus/router.py` + `bus/bridge.py` 路由)

| B→A event `tool` | `to` | A 侧分发(后端 A spec §3.4) |
|---|---|---|
| `pose` | `"alpha"` | 更新 `current_pose`(AppState)+ 转发前端 WS `pose` + 进遥测缓冲;α loop 读作翻译上下文 |
| `telemetry` | `"alpha"` | **实现**(A8,见下方分帧约定):10Hz 富 IMU,进遥测缓冲(每秒批量入库)+ α loop 读;**不推前端** |
| `status` | `"alpha"` | 更新 `flight_status` + 转发前端 WS `status`(含当前动作/总动作进度,替代航点段) |
| `reject` | `"alpha"` | **小模型/ego-planner 无法生成可达目标**(未知动作编码、夹紧后越界、阶段2/4 ego-planner 不可达);注入 α 上下文(下轮重想或兜底 hover)+ 转发前端 WS `reject` |
| `alert` | `"beta"` | monitor-trigger 处理:WS 推前端 `alert`(β 系统消息)+ 注入 β 上下文唤醒 β 给处置建议(不自动执行) |
| `pong` | **心跳特例(`to="heartbeat"`,A 侧 IPC 层直接消费)**:在 A 侧 `ipc/server.py` 层直接 `update 心跳时间戳`,**不进组件路由表**;`to` 字段冻结取值 `"heartbeat"`(两侧一致,A18),否则视为协议错误记日志丢弃 |

> `pong` 是唯一不进 A 侧组件路由表的 event。其余 5 类必须 `to` 组件名精确命中,否则视为协议错误记日志丢弃(B spec §8 "msgpack 解帧失败 → 丢弃不断连"的同源处理)。
>
> **pose/telemetry 分帧约定(对齐前端 §3.3 PoseMessage,A8 实现)**:B 侧 **10Hz 同时发 `pose` 与 `telemetry` 两帧**——`pose` payload 含 `pos`/`quat`/`vel`/`accel`/`angularVel`(与前端 `PoseMessage` 1:1,A 转推前端);`telemetry` 携带富 IMU(原始加速度/角速度等,与前端无关),**仅入 A 侧遥测缓冲/历史库,不推前端**(前端无独立 telemetry 消息)。A 侧历史 `telemetry` 表从 `pose`+`telemetry` 合并填入(§5.1 字段 `quat_w/x/y/z`/`accel_*`/`angular_velocity_*`),与总体架构 §5.1 对齐。
>
> **`pose` payload 字段顺序(冻结,防四元数歧义)**:`pos`=`[x,y,z]`、`quat`=`[w,x,y,z]`(**实部在前**,对齐前端 §3.3 行 269 + 前端 state §2.1 行 103)、`vel`=`[vx,vy,vz]`、`accel`=`[ax,ay,az]`、`angularVel`=`[wx,wy,wz]`。B 侧 `rosbridge/adapter.py` 从 ROS `geometry_msgs/PoseStamped`(`qx,qy,qz,qw`)构造 `quat` 时**必须重排为 `[w,x,y,z]`**,否则前端 yaw 解析会错。

## 4. A→B `call.tool` 枚举(冻结,schema_version=2)

| `tool` | `args` | 含义 | B 侧处理 |
|---|---|---|---|
| `action` | **ActionCommand**(总体架构 §2.2 格式,`task_id`/`actions`/`safety_constraints`)| 下发动作编码(α 产出,替代旧 `trajectory`)| `bus.call(to="small_model", tool="generate_goal", args={action,pose,env})` |
| `abort` | `{}` | 中止当前动作序列 | `bus.call(to="small_model", tool="abort")` |
| `hover` | `{}` 或 `{pose?}` | 安全悬停 | `bus.call(to="small_model", tool="hover")` |
| `ping` | `{}` | 心跳(2s) | B 侧 `ipc` 层直接回 `pong`,不下发 small_model |

> **scope 标注**:此 4 项是 A↔B 跨进程 IPC 的 call.tool **全集**。A 内总线的 `alpha.translate`(β 调,后端 A spec §3.5)**不跨 A↔B**,不在此冻结;B 内总线组件间的 `tool` 同理不在此冻结。
>
> **2026-07-07 重构**:原 `trajectory`+TrajectorySpec 工具废弃,改 `action`+ActionCommand;`SCHEMA_VERSION` 升 1→2,两侧 `bus/protocol.py` 同步。

> **`result` 暂不强制跨进程回传**:S3 联调先以 fire-and-forget 下发为主;若后续需要 A 侧确认 B 收到,再启用 `result`(`call_id` 配对),届时升级 `SCHEMA_VERSION` 而非静默加。先导实现 B 侧可立即回 `result`(B spec §2.2 small_model 组件产出 `result`),A 侧不强依赖。

## 5. B→A `event.tool` 枚举(冻结)

| `tool` | 频率 | `payload` 关键字段 | `to` | 前端呈现(A 转发后) |
|---|---|---|---|---|
| `pose` | 10Hz | `{pos:[x,y,z], quat:[w,x,y,z], vel:[vx,vy,vz], accel:[ax,ay,az], angularVel:[wx,wy,wz], ts}`(字段顺序冻结,见 §3 注) | `"alpha"` | WS `pose`(前端 §3.3 PoseMessage) |
| `telemetry` | 10Hz,**实现**(见 §3 分帧约定,A8) | `{vel, accel, imu, ts}` | `"alpha"` | **不推前端**(入 A 侧遥测缓冲/历史库) |
| `status` | 低频 | `{flightStatus, mode, currentAction, totalActions, taskId}`(动作进度,替代航点段) | `"alpha"` | WS `status`(前端 §3.3 StatusMessage) |
| `reject` | 事件 | `{reason, actionIndex, suggestedAction?}`(小模型/ego-planner 不可达) | `"alpha"` | WS `reject`(前端 §3.3 RejectMessage) |
| `pong` | 2s 响应 ping | `{}` | 不强约束(心跳特例) | 不推前端 |
| `alert` | 检测周期(10Hz)+ 节流 | `{level, code, detail, suggestion?, ts, action_index?}` | `"beta"` | WS `alert`(前端 §3.3 AlertMessage,作 β 对话流系统消息)+ 唤醒 β |

> **不在 A-B 接口的 WS 消息(避免实现者误解)**:`alpha_output`、`video_frame`、`voice_tts`、`voice_stt_result`、`sync_response`、`link_status` 是 **A→前端** 的 WS 消息,**不经 A-B IPC**,B 侧不产。其中:
> - `alpha_output`/`link_status` 由 A 侧 α loop / IPC 层产生(总体架构 §3.3、A spec §4.3、冻结 §6);
> - **`video_frame` 先导期不实现**(A4):阶段1 ROS 假无人机无摄像头,A 不下发;阶段2 PX4 + 相机接入后由 A 侧音视频模块产生。**与语音无关**,原把 `video_frame` 与 `voice_*` 并列是笔误。
> - `voice_tts`/`voice_stt_result`/`sync_response` 由 A 侧语音与同步模块产生。
> B 上行只冻结本表 6 类 event。

> **alert 安全边界**(总体架构 §1.3、§2.5;B spec §6.4):B 只产 alert,**不自动执行飞行处置**;β 处置建议需人确认才(经 `propose_to_alpha`)执行。`suggestion` 字段 B 侧置空,由被唤醒的 β 填。
>
> **alert 节流**(B spec §6.4):同 `code` 在 N 秒内只上告一次(默认 2s),`critical` 级别不节流。

## 6. 心跳与断连(冻结,对齐总体架构 §3.6 + B spec §2.3)

| 项 | 冻结值 |
|---|---|
| ping 周期 | A 每 **2s** 发 `call.ping` |
| pong 响应 | B 收 ping 立即回 `event.pong` |
| A 侧断连判定 | 超 **5s** 无 pong → 视为 B 断连 → A 前端 WS `link_status{link:"A-B", state:"down"}` |
| B 侧断连处理 | socket 断开 → `ipc_connected=False` → small_model 切 `hover`(用 BState 当前位姿)→ 每 1s 重连 |
| 重连后 | A、B 双方不假设对方状态;B 重连后等 A 下发新意图(不发历史回放);A 收到新 pose 即刷新 AppState |

## 7. 关停顺序(冻结,对齐 B spec §3.3 + A spec §6.2)

**A 关停**:经 IPC 给 B 发 `call.hover` → 关 IPC server → B 收到断连后维持悬停。
**B 关停**:small_model 切 `hover`(下发最后意图)→ 关 ROS publisher(末帧悬停)→ `rospy.signal_shutdown()` → 关 IPC socket。
**任何路径下,最后下发的 setpoint 必须是悬停**(总体架构 §3.6)。
