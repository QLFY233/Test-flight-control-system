# 试飞控制系统 — 总体架构设计

> 先导项目:打通"大模型理解试飞意图 → 翻译为无人机目标点 → 求解器生成安全轨迹 → 模拟无人机执行"完整链路。
> 本阶段无真实无人机,用 ROS1 模拟无人机代替。

| 组件 | 对外接口(别人怎么用它) | 可独立改的内部 |

| 前端 | 只认 A 一个地址(Web 协议) | UI 布局、3D/2D 库 |

### 1.3 安全边界

- 大模型绝不直接碰飞控:α 只产出"点序列",求解器做物理校验(场地内?会撞?速度合理?)才下发。
- β 不直连求解器:β 产出自然语言计划,经人审核、α 翻译,才到求解器。
- A↔B 的 Unix socket 接口是天然安全检查点。

### 1.4 功能范围

| 维度 | 决策 |
|---|---|
| 场地 | 室内先导(室外远期) |
| 场地建模来源 | 人工定义配置文件(YAML) |
| 单/多机 | 单机 |
| 环境条件(天气/风速/湿度) | 前端手动输入,提供预设模板和历史记录复用 |
| 任务类型 | 自由任务(自然语言描述,β 自由规划) |
| 部署 | 单机,内网 HTTP,WSL2 mirrored 模式 |

## 2. 数据流与三层翻译链

### 2.1 三层翻译链

```
自然语言任务 → β → 自然语言飞行计划 → 人审核 → α → 点序列(带逐段约束) → 求解器 → 连续轨迹 → PX4
```

| 层 | 执行者 | 输入 | 输出 | 工具/依赖 | 频率 |
|---|---|---|---|---|---|
| L1 规划 | Agent β | 自然语言任务描述 | 自然语言飞行计划(人可读) | `get_field_map`/`get_current_pose`/`query_history` 等工具 | 按需,低频 |
| L1.5 审核 | 人 | β 的自然语言计划 | 确认/修改后的计划 | β 界面 | 按需 |
| L2 翻译 | Agent α | 审核后的自然语言计划 + 键盘实时输入 | 点序列(带逐段约束 JSON) | 读当前位姿/场地(只读) | loop 持续,秒级 |
| L3 求解 | 求解器(B 内) | 点序列(带逐段约束) | 连续轨迹(位置/速度/角速度/姿态时变曲线) | 场地配置(防碰撞) | 10~50Hz |
| L4 执行 | PX4/假无人机 | 连续轨迹期望(setpoint) | 实际飞行 | ROS 话题 | 毫秒级 |

### 2.2 全链路约束体系

三层约束,后者覆盖前者:全局 → 段 → 点。约束值来源:全局默认配置 + α 按需覆盖。

```json
{
  "task_id": "20260630-001",
  "schema_version": 1,
  "global_constraints": {
    "keep_clear_distance": 0.5,
    "ceiling": 2.5,
    "floor": 0.3
  },
  "segments": [
    {
      "waypoints": [
        {
          "position": [3.0, 2.0, 1.0],
          "yaw": 0.0,
          "constraints": {
            "speed_max": 1.2,
            "accel_max": 2.0,
            "angular_velocity_max": 0.5
          }
        },
        {
          "position": [3.5, 2.5, 1.0],
          "constraints": {"speed_max": 1.0}
        }
      ],
      "segment_constraints": {
        "speed_max": 1.2,
        "accel_max": 2.0,
        "angular_velocity_max": 0.5
      }
    },
    {
      "waypoints": [
        {
          "position": [0.0, 0.0, 0.5],
          "return_home": true
        }
      ],
      "segment_constraints": {"speed_max": 0.8, "accel_max": 1.0}
    }
  ]
}
```

约束层级:点约束 > 段约束 > 全局约束 > 求解器默认。
可扩展:加新约束类型就是加新 key(如 `jerk_max`/`attitude_limit`/`time_window`)。求解器用约束注册表:认得的执行,不认得的忽略并告警。

### 2.3 α 双重职责与 loop 仲裁

α 在 loop 内承担两个来源,都产出"点":

- 来源 i:人类实时输入(键盘离散操控 + 语音指令)。人通过键盘或语音发出"向上/向下/平移/悬停",α 翻译成增量目标点,**放进下一个 loop 周期**(单步内不抢占)。
- 来源 ii:审核后的自然语言计划。α 翻译成多段点序列,作为任务注入 loop,求解器逐段执行。

**loop 仲裁逻辑:**

```
α loop 周期开始
  ├ 读取:当前位姿(从 B 上行)、当前执行中的预设段、人类输入队列(键盘+语音)
  ├ 仲裁规则:
  │   if 人类输入队列非空:
  │       本周期下发人类输入翻译的点(打断预设)
  │   elif 执行中预设还有未完成段:
  │       本周期下发下一段目标点
  │   else:
  │       本周期下发"保持悬停"(安全默认)
  ├ 翻译:自然语言/键盘/语音 → 点(调用 LLM)
  └ 输出:点(经 A↔B IPC 发给 B 求解器)
周期结束,人类新输入累入下个周期
```

**打断规则:**
- loop 单步内:人类输入(键盘+语音)排队到下一周期(不抢占当前周期)。
- 预设轨迹整体:可被人类输入打断(人优先,随时接管)。被打断的预设剩余部分丢弃或暂停。
- 安全默认:无任务、无人输入时 α 下发"悬停",飞控总有合法期望值。

### 2.4 β 工具集

| 工具 | 输入 | 输出 | 数据源 |
|---|---|---|---|
| `get_field_map()` | 无 | 场地边界+障碍列表 | 场地配置文件 |
| `get_current_pose()` | 无 | 当前位姿/速度 | B 上行状态(A 缓存) |
| `query_sessions(filter)` | 时间/任务等过滤 | 试飞会话列表 | 历史库 |
| `query_telemetry(session_id, range)` | 会话ID+时间范围 | 轨迹时序点 | 历史库 |
| `query_environment(env_id)` | 环境ID | 环境条件(JSON) | 历史库 |
| `query_conversations(session_id)` | 会话ID | 对话记录 | 历史库 |

### 2.5 双 Agent 协作(已定)

```
β 界面(分析/规划)                      α 界面(实时操控)
       │                                      │
       ▼                                      │
   Agent β                                     │
   调历史工具 → 产出自然语言飞行计划        ┌───┘
       │                                   │
       ▼                                   │
   人审核(确认/修改/驳回)                   │
       │ 确认后                             │
       └───────────────────────────────→ Agent α
                                          翻译 → 点序列 → 求解器 → PX4
```

- β 与 α 不直接通信,唯一数据通道是"β 产出 → 人审核 → 注入 α"。
- 各自独立界面、各自独立 Pydantic AI Agent 实例、各自独立上下文。
- β 上下文偏长(历史+规划),α 偏短(当前状态+任务+键盘)。
- 先导模型:DeepSeek-V4-Flash(OpenAI 格式,经 Pydantic AI openai provider)。

## 3. 通信 fabric 与协议契约

### 3.1 链路总表

| # | 链路 | 协议 | 方向 | 频率 | 消息格式 |
|---|---|---|---|---|---|
| L1 | 前端 ↔ A | WebSocket | 双向 | 10Hz~视频 | JSON |
| L2 | 前端 ↔ A | SSE | A→前端 | 秒级(流式) | Pydantic AI UIAdapter 事件 |
| L3 | 前端 ↔ A | REST | 双向 | 按需 | JSON |
| L4 | 前端 ↔ A | StaticFiles | A→前端 | 一次 | HTML/JS/CSS |
| L5 | A ↔ B | Unix domain socket | 双向 | 上行10Hz/下行秒级 | 上行 MessagePack,下行 JSON |
| L6 | B ↔ 假无人机/PX4 | ROS 话题 | 双向 | 毫秒~10Hz | ROS msg |
| L7 | β/A ↔ 历史库 | SQLAlchemy | 双向 | 按需 | SQL |
| L8 | β/A/求解器 ↔ 场地配置 | 文件读 | 单向 | 按需 | YAML |
| L9 | A ↔ DeepSeek | HTTPS | 双向 | 秒级 | OpenAI 格式(经 Pydantic AI) |
| L10 | A ↔ 讯飞 STT/TTS | HTTPS | 双向 | 按需 | 讯飞 API(细节待补) |

### 3.2 L5:A↔B Unix socket 契约

消息帧: `[4字节长度][载荷]`(length-prefixed,防止粘包)。

- 下行(A→B,低频):JSON 文本
- 上行(B→A,高频10Hz):MessagePack 二进制

| 方向 | type | 含义 |
|---|---|---|
| A→B | `trajectory` | 点序列(带约束)下发 |
| A→B | `abort` | 中止当前轨迹(键盘打断预设) |
| A→B | `hover` | 安全悬停(无任务) |
| A→B | `ping` | 心跳/存活检测 |
| B→A | `pose` | 当前位姿(高频) |
| B→A | `telemetry` | 速度/加速度/IMU(高频) |
| B→A | `status` | 求解器/无人机状态(低频) |
| B→A | `reject` | 求解器拒绝某段,回告 α 重想 |
| B→A | `pong` | 心跳回应 |

### 3.3 前端↔A 契约

**SSE(流式大模型文字):** α/β 各一个端点(`/chat/alpha`、`/chat/beta`),Pydantic AI UIAdapter 返回 `StreamingResponse`。

**WebSocket(实时数据):**

| 方向 | 消息示例 | 用途 |
|---|---|---|
| 前端→A | `{type:"keyboard", key:"up", ts:...}` | 键盘操控 |
| 前端→A | `{type:"voice_frame", data:<base64>}` | 语音输入(待讯飞API定) |
| A→前端 | `{type:"pose", pos:[...],quat:[...],vel:[...]}` | 实时位姿(3D轨迹) |
| A→前端 | `{type:"video_frame", data:<base64>}` | 视频回传 |
| A→前端 | `{type:"status", segment:2, total:5}` | 任务进度 |
| A→前端 | `{type:"voice_tts", data:<base64>}` | 语音朗读下行 |

**REST:** `GET/POST /history/sessions`、`/history/telemetry/{id}`、`/overview`、`/environments`。

**语音:** STT/TTS 均调讯飞 API。音频传输方式待讯飞 API 文档后再定。

### 3.4 L6:ROS 话题契约

**阶段1(纯 ROS 假无人机):**

| 话题 | 类型 | 方向 |
|---|---|---|
| `/drone/setpoint_position/local` | `PoseStamped` | B→无人机 |
| `/drone/setpoint_velocity/cmd_vel` | `Twist` | B→无人机 |
| `/drone/local_position/pose` | `PoseStamped` | 无人机→B |
| `/drone/local_position/velocity` | `TwistStamped` | 无人机→B |
| `/drone/imu/data` | `Imu` | 无人机→B |

**阶段2(PX4 SITL + mavros):** 话题前缀换成 `/mavros/...`,消息类型不变。求解器对下游用统一话题接口(配置前缀),换实现只改配置。

### 3.5 序列化版本

所有跨进程 JSON 加 `schema_version` 字段。MessagePack 上行帧同带版本号。

### 3.6 失败/断连处理

| 场景 | 处理 |
|---|---|
| A↔B socket 断开 | B 维持最后位姿安全悬停,A 告警前端 |
| A 崩溃 | B 收不到新意图,维持安全悬停 |
| B 崩溃 | A 检测断连告警;无人机 ROS 节点独立可停 |
| 大模型 API 失败 | α/β 当前回合失败,不下发新意图(维持悬停),重试/告警 |
| 求解器 reject | A 收到,α 重新翻译或告警人 |

核心原则:任何环节失联/失败,无人机进入安全悬停。

## 5. 历史数据 schema + 场地配置格式

### 5.1 数据库表结构(SQLAlchemy ORM)

```sql
-- ① 环境条件(JSON 灵活字段)
CREATE TABLE environments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    data TEXT NOT NULL,   -- JSON: {"weather":"晴","wind":[0.2,0.0,0.1],"humidity":45,...}
    created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- ② 试飞会话
CREATE TABLE flight_sessions (
    id TEXT PRIMARY KEY,                -- "20260630-001"
    created_at TEXT NOT NULL,
    environment_id INTEGER REFERENCES environments(id),
    task_description TEXT,
    beta_plan TEXT,                     -- β 产出的自然语言计划
    alpha_trajectory TEXT,              -- α 翻译的点序列 JSON
    status TEXT DEFAULT 'planned'       -- planned/executing/completed/aborted
);

-- ③ 轨迹时序(高频)
CREATE TABLE telemetry (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT NOT NULL REFERENCES flight_sessions(id),
    t REAL NOT NULL,
    position_x REAL, position_y REAL, position_z REAL,
    velocity_x REAL, velocity_y REAL, velocity_z REAL,
    accel_x REAL, accel_y REAL, accel_z REAL,
    angular_velocity_x REAL, angular_velocity_y REAL, angular_velocity_z REAL,
    quat_w REAL, quat_x REAL, quat_y REAL, quat_z REAL,
    UNIQUE(session_id, t)
);
CREATE INDEX idx_telemetry_session ON telemetry(session_id);

-- ④ 对话记录
CREATE TABLE conversations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT NOT NULL REFERENCES flight_sessions(id),
    agent TEXT NOT NULL,                -- 'alpha' / 'beta'
    role TEXT NOT NULL,                 -- 'human' / 'agent' / 'tool_call' / 'tool_result'
    content TEXT NOT NULL,
    metadata TEXT,                      -- JSON: {tool_name, tool_args, ...}
    created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX idx_conv_session ON conversations(session_id);
```

关键设计:
- `environments.data` 用 SQLAlchemy `JSON` 列类型(SQLite→JSON,Postgres→JSONB),查询用 ORM JSON 操作。
- `telemetry` 用 `(session_id, t)` UNIQUE + 索引。
- `conversations.agent` 区分 α/β,`role` 区分人/agent/工具调用/返回。

### 5.2 数据写入时机

| 事件 | 写入 |
|---|---|
| 试飞开始 | INSERT `flight_sessions`(status=planned) |
| 环境手动输入 | INSERT/UPDATE `environments` |
| α 产出点序列 | UPDATE `flight_sessions.alpha_trajectory` |
| 飞行中(10Hz) | 批量 INSERT `telemetry`(缓冲,每秒一批) |
| α/β 每轮对话 | INSERT `conversations` |
| 试飞结束 | UPDATE `flight_sessions.status` |

求解器碰撞检测:

| 形状 | 方法 |
|---|---|
| `box` | 轨迹段与 AABB/OBB 求交 |
| `cylinder` | 段到圆柱轴心线距离 ≤ radius |
| `sphere` | 段到球心距离 ≤ radius |
| `plane` | 轨迹点是否在法向禁止侧 |
| `capsule` | 段到胶囊体轴线距离 ≤ radius |

可扩展:加新形状 = 加 type + 对应参数 + 求解器加碰撞函数。先导实现 box/cylinder/sphere/plane,capsule 后补。
