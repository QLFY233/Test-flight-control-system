电脑版：
整个页面左右分栏左右比例不固定，不同情况对应调节，主要根据右栏视频尺寸调节，，左边分上下三两栏，上栏时状态模拟，有湿度，风速等选项，下栏是与大模型对话界面，右边是视图，自适应视频尺寸，视频源优先度：三维轨迹图>视频回传>二维图表，在右栏上方，可以自由选择排列数量（一个，两个，三个），选择只放一个视频源时,默认按有优先度排列，存在切换视频源按钮，排列两个时，上下排列，自主选择视频源排列，点击后默认按有优先度排列。

整体主题以黑色为主，按钮和主题做分层处理，按钮边框要圆角

存在一个悬浮球，点击后跳出快捷键选项，支持自定义快捷键，点击自定义快捷键后会记录飞机上次操作，自定义操作可以主动取消，有些快捷键存在二次选择，比如上升快捷键，点击后选择上升多少米，选择界面以小视图存在，点击后不对其他操作造成影响。

新增功能（2026-07-01 更新）:

- **三维视图交互**：右侧三维轨迹视图需要支持鼠标拖动（旋转）、滚轮缩放、右键/双指平移等交互。优先采用 Three.js + OrbitControls 实现，默认启用轨道控制（旋转/缩放/平移）。三维视图依旧享有最高展示优先级，用户可在右栏上方选择显示数量（1/2/3），单个显示时优先展示三维轨迹图。

- **语音识别按钮**：在与大模型对话输入框的旁边加入语音识别按钮（麦克风图标）。点击后前端仅采集音频，通过 WebSocket 发送 `{type:"voice_frame"}` 至后端 A，由后端调用讯飞 STT API 识别，识别结果经 SSE 回填输入框（可自动发送，可选）。需提供识别状态指示（采集中/识别中/完成/错误）。移动端同样在输入旁展示该按钮。

- **历史记录与 β 界面（新增）**：新增一个 Beta（β）界面与历史记录界面。总体交互：左侧为历史记录与筛选区，右侧为 β（并可切换到 α）——整体分为两个 TAB：`α` 与 `β`；每个 TAB 内部右侧显示对应模型对话与规划视图。
	- 左侧区域分成两个子 TAB：
		- **按飞行次数（Flight Tabs）**：每次飞行作为一个子 tab，列出该次飞行的记录（时间戳、摘要、关键轨迹预览）。
		- **按数据类型（By Data Type）**：按数据类别（遥感图像、传感器时序、轨迹数据、视频回传等）过滤和查看记录。
	- 用户可以在左侧进行多重筛选（时间区间、数据类型、关键字），筛选结果以列表形式展示；点击某个记录后，右侧（β 区域）显示该记录的**建模结果**和**β 对话/规划窗口**。
	- 在 β 进行飞行路径规划时，右侧的 β 界面必须能同时显示对应的建模结果（数值 / 图表 / 模型摘要），并能将规划结果叠加到三维视图（右栏）中以供预览与验证。
	- 在历史记录界面提供“发送到 β”按钮：选择一个或多个记录，点击后将记录内容发送给 β，会在 β 的会话历史中产生新的上下文条目。

- **桌面/移动 UI 一致性**：桌面版与手机版需要包含相同的主体功能（3D 视图、语音输入、历史记录检索、α/β TAB、建模展示、β 规划叠加等）。手机版的布局采用你现有的手机 UI 修改：上下两栏（上：视频/三维视图可滑动切换；下：对话与输入），但必须把上述新增功能（语音按钮、历史检索、β 建模显示、路径叠加）整合进移动界面。

实现建议与注意事项：

- 三维视图性能：移动端需根据设备能力降级（减少粒子数、简化轨迹），并提供 "低性能模式" 切换。
- 数据流：历史记录筛选后发送到 β 时，建议通过后端 API 以 JSON 包（metadata + 小样本数据）发送，前端提供预览与确认界面。
- 无障碍与反馈：语音识别需要在识别中/识别结束/错误时提供明显的 UI 反馈（红色/绿色指示），并允许用户手动编辑识别结果。
- 可测试的 prototype：建议先用一个本地静态 prototype（Three.js + Web Speech API + 假数据）验证交互流程，再接入真实后端。
