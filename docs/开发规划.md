## 阶段A — 基础设施与协议常量（硬前置，阻塞一切）

> **理由**：A↔B 双进程协议常量必须两侧逐字一致才能联通；msgpack 跨 0.6/1.x 版本是硬契约。先把这些"地基"一次铺好，后续两侧可并行编码不冲突。

### A.1 环境与 venv

| 任务 | 命令/文件 | 验收 |
|---|---|---|
| 装 Python 3.10 (deadsnakes) | `sudo add-apt-repository ppa:deadsnakes/ppa && sudo apt install python3.10 python3.10-venv python3.10-dev` | `python3.10 --version` |
| 创建 venv-A | `python3.10 -m venv .venv-A`；pip 装 FastAPI 0.136.3 / Pydantic AI 2.0.0 / SQLAlchemy 2.0.51 / Uvicorn 0.40.0 / aiosqlite / msgpack(pip 最新) / websockets | `import fastapi, pydantic_ai, sqlalchemy` 全 OK |
| 装 B 侧系统 apt | `sudo apt install python3-msgpack python3-scipy` (msgpack 0.6.2 / scipy 1.3.3) | `apt-cache policy python3-msgpack` = 0.6.2-1 |
| 创建 venv-B（`--system-site-packages`!） | `python3 -m venv --system-site-packages .venv-B`；pip 装 pyyaml | 见下方冲突检查 |
| 启动脚本 | `run_backend_b.sh`：`source /opt/ros/noetic/setup.bash` **先** → `source .venv-B/bin/activate` **后** → `exec python -m backend_B.main` | 脚本可执行 |

### A.2 仓库骨架与配置文件

按总体架构 §4.4 一次性建好目录：`backend-A/ backend-B/ sim-drone/ frontend/ config/ data/ docs/ .env.example`（真实 `.env` gitignore）。

| 文件 | 内容 |
|---|---|
| `config/field.yaml` | 先导收缩版：仅 `boundary` + `home`，`obstacles` 整段删除（总体架构 §5.3） |
| `config/default_constraints.yaml` | `speed_max 1.5 / ceiling 2.5 / floor 0.3 / accel_max 2.0 / angular_velocity_max 0.5`；**`keep_clear_distance` 删除**（§5.4） |
| `venv-A-requirements.txt` / `venv-B-requirements.txt` | 锁定版本（对齐总体架构 §4.2） |
| `.env.example` | `LLM_PROVIDER=deepseek` / `DEEPSEEK_API_KEY=` / `XF_APP_ID=` 等（空值入库） |

### A.3 协议常量（两侧同时写，逐字一致）

| 文件 | 内容 |
|---|---|
| `backend-A/bus/protocol.py` & `backend-B/bus/protocol.py` | `SCHEMA_VERSION = 2`；`MSGPACK_USE_BIN_TYPE = True`；`msg_type` 常量；`to` 名常量（`alpha/beta/small_model/ego_planner/lidar/monitor/heartbeat/broadcast`） |
| `backend-A/ipc/frames.py` & `backend-B/ipc/frames.py` | `send_frame/recv_frame`：4 字节大端长度前缀 + msgpack；**Packer 显式 `use_bin_type=True`**；unpackb `raw=False, use_bin_type=True` |

### ⚠️ 阶段A 冲突/约束检查清单

| # | 风险 | 检查方法 |
|---|---|---|
| 1 | msgpack 0.6(B) vs 1.x(A) bin/str 错位 | S0 验证：两端各跑 `packb(b"x", use_bin_type=True)`+`unpackb(...)`，确认互解无类型错位才进下一阶段（B spec §12） |
| 2 | venv-B 不加 `--system-site-packages` → rospy/numpy 不可见 | `source ROS && activate && python -c "import rospy, numpy, msgpack"` 全成功 |
| 3 | source 顺序错（venv 在前 ROS 在后）→ PYTHONPATH 不对 | `run_backend_b.sh` 严格"先 ROS 后 venv" |
| 4 | SCHEMA_VERSION 两侧不一致 | 用同一份 `protocol.py` 拷贝或符号共享，git 一次提交两侧 |
| 5 | `keep_clear_distance` / `obstacles` 残留 | grep 全仓库确认无残留（废弃概念，阶段2/4 交 ego-planner） |

**✅ S0 验收门**：msgpack 帧 A↔B 互解通过 → 进入阶段 B/C/D。

## 阶段B — 后端 B 脊柱（bus + IPC + BState）

> **理由**：后端 B 是飞控执行端，先建好它的"骨架 + 上行/下行消息管道"，后续小模型/监控/ROS 桥都往这个骨架上挂。与阶段C（A 侧脊柱）**完全并行**，互不阻塞。

### B.1 BState 与配置加载

| 文件 | 内容 |
|---|---|
| `backend-B/state.py` | `BState`：`current_pose/current_velocity/current_imu/last_data_ts/current_goal/current_action_index/current_action_plan/ipc_connected/small_model_status/field/default_constraints`；`pose_lock` 独立保护高频字段 |
| `backend-B/config_loader.py` | 校验 `field.yaml` + `default_constraints.yaml`，失败即退出 |

### B.2 B 内内存总线

| 文件 | 内容 |
|---|---|
| `backend-B/bus/registry.py` | 静态注册：`small_model`(tool: `generate_goal/abort/hover`)、`monitor`；`ego_planner/lidar` 阶段2/4 不注册 |
| `backend-B/bus/router.py` | `bus.call(to, tool, args)` 同步查表调组件返 result |

### B.3 A↔B IPC client

| 文件 | 内容 |
|---|---|
| `backend-B/ipc/client.py` | 连 `/tmp/flight_control_AB.sock`；**恒定时间重连每 1s**（非指数退避）；socket 断 → `ipc_connected=False` |
| `backend-B/ipc/dispatch.py` | A→B call 解帧进 B 总线；B→A event 封帧上行；心跳：收 `ping` 立即回 `pong` |

### ⚠️ 阶段B 冲突检查

| # | 风险 | 对策 |
|---|---|---|
| 1 | B 侧不主动判 A 断连（A 判，B 被动） | B 只在 socket 断开时 `ipc_connected=False`；心跳超时由 A 算 |
| 2 | `pose` payload `quat` 字段顺序错 | 冻结 `[w,x,y,z]` 实部在前；adapter 从 ROS `qx,qy,qz,qw` **必须重排**（接口冻结 §3） |

## 阶段C — 后端 A 脊柱（bus + IPC server + AppState + DB）

> **理由**：与阶段B 并行。A 侧 bus/IPC server 先建好，就能在阶段G 接 α、阶段H 接 β。DB 与静态托管是叶子，顺手做完。

### ⚠️ 阶段C 冲突检查

| # | 风险 | 对策 |
|---|---|---|

## 阶段E — 假无人机（独立 catkin 包，只依赖 ROS）

> **理由**：`sim-drone/` 不属于 backend-B，只依赖 ROS，可独立编码并 S1 单跑验证。把它前置，让阶段F 的 B 侧有东西可以通信。

| 文件 | 内容 |
|---|---|
| `sim-drone/CMakeLists.txt` + `package.xml` | `<depend>rospy/geometry_msgs/sensor_msgs</depend>`；`catkin_install_python(PROGRAMS scripts/fake_drone_node.py)` |
| `sim-drone/scripts/fake_drone_node.py` | 订阅 `/drone/setpoint_position/local`；**运动学积分**（线性插值，速度上限 2 m/s）；50Hz 发 pose/velocity/imu；**超时悬停 >500ms 无 setpoint**；**边界自保：目标超 boundary 停在边界内最近点** |
| `sim-drone/launch/fake_drone.launch` | `<node name="fake_drone" pkg="sim_drone" type="fake_drone_node.py" output="screen"/>` |

`cd ros_ws && catkin_make && source ros_ws/devel/setup.bash`。

### ⚠️ 阶段E 冲突检查

| # | 风险 | 对策 |
|---|---|---|
| 1 | 假无人机无超时悬停 → B 崩溃时乱飞 | **必须实现 >500ms 无 setpoint 自动悬停**（B spec §9.1） |

**✅ S1 验收门**：`roscore` + `rosrun sim_drone fake_drone_node.py` + 手动 `rostopic pub` → 假无人机移动并回传位姿 + 超时悬停。

## 阶段F — B 侧 small_model stub + ROS 桥（打通 S2）

> **理由**：有了假无人机 + B 脊柱，现在把"动作编码 → 目标点 → setpoint"这条最短链跑通。**这是整个系统的第一个端到端可验证里程碑**——hardcoded ActionCommand 直接喂 small_model。

### F.1 small_model stub（先导规则占位，**不训练**）

| 文件 | 内容 |
|---|---|
| `backend-B/small_model/action_codes.py` | 9 类编码常量 |
| `backend-B/small_model/goal_gen.py` | `GoalGenerator` ABC + `make_goal_generator()`（env `SMALL_MODEL_BACKEND=stub\|onnx`） |
| `backend-B/small_model/stub.py` | 9 类规则映射 + boundary/ceiling/floor/speed_max 夹紧；未知编码抛 `GoalGenError("unknown_action_code")` |
| `backend-B/small_model/component.py` | 接 `generate_goal/abort/hover`；成功产目标点 + status；失败产 `event:reject` |

### F.2 ROS 桥（阶段1 假无人机）

| 文件 | 内容 |
|---|---|
| `backend-B/rosbridge/topics.py` | 阶段1 前缀 `/drone/`；话题表见总体架构 §3.4 |
| `backend-B/rosbridge/node.py` | `rospy.init_node("backend_b", anonymous=False)` |
| `backend-B/rosbridge/publisher.py` | 目标点线程 20Hz；**首帧前填一个当前位置 setpoint** 防跳变；按 `speed_max` 限速；到达阈值切下一条动作 |
| `backend-B/rosbridge/subscriber.py` | 订阅位姿/速度/IMU → 加锁写 BState；维护 `last_data_ts` |
| `backend-B/rosbridge/adapter.py` | `Phase1Adapter`；阶段2 `Phase2Adapter`（**阶段M 才写**） |

### F.3 线程模型 + 生命周期

`backend-B/lifecycle.py` 启动 N 步：配置→BState→ROS 节点→B 总线→IPC client→**先连 A 再启目标点线程**→uplink 10Hz→rospy.spin。线程：rospy 主 + 目标点(20Hz) + uplink(10Hz) + ipc。**不引入 asyncio**。

### ⚠️ 阶段F 冲突检查

| # | 风险 | 对策 |
|---|---|---|
| 1 | 引入 asyncio 与 rospy 冲突 | B 侧**只用 threading**（B spec §3.1） |
| 2 | 首帧 setpoint 从 (0,0) 跳变 | publisher 首帧填当前位置 |
| 3 | B 启动时 A 未连上就跑 setpoint | **先连 A 再启目标点线程**；A 未连上 small_model 维持悬停 |
| 4 | stub 误用 numpy/scipy 插值 | 先导 stub **不引入**轨迹插值（B spec §4.5） |
| 5 | onnxruntime 误装 | 先导 stub **不装 onnxruntime**，纯 Python 规则 |

**✅ S2 验收门**：B 单跑 + 假无人机响应 + uplink 自验。

## 阶段G — A↔B IPC 通 + α Agent（S3 + S5 前半）

> **理由**：阶段B/C 铺好两侧 IPC，阶段F 让 B 能消化 ActionCommand。现在 A↔B 握手（S3）然后接 α。

### G.1 A↔B IPC 联通（S3）

A 发 `ping` → B 回 `pong`；A 发 hardcoded `action` call（不经 α）→ B small_model 执行 → 假无人机动；A 收 B 上行 `pose`(10Hz)、`status`。

### G.2 α Agent（LLM 翻译器）

| 文件 | 内容 |
|---|---|
| `backend-A/agents/llm.py` | `make_agent`；`PROVIDERS={"deepseek":{...model:"deepseek-v4-flash"}}`；`OpenAIModel` |
| `backend-A/agents/translator_base.py` | `ActionTranslator` ABC；`TranslateError` |
| `backend-A/agents/alpha_llm.py` | `LLMTranslator` 实现 |
| `backend-A/agents/alpha.py` | `make_translator()` 按 `ALPHA_BACKEND=llm\|small`；α loop 仲裁；**`await translate` 用 `asyncio.to_thread`**；失败 → hover |
| `backend-A/prompts/alpha.md` | α 系统 prompt（翻译器，**不对话**）；约束 `schema_version=2` |

### G.3 conversations 自动写（带训练标的标志）

`run_agent_with_log` 写 α 的 `tool_call` 时 `metadata` **必含 `approved` + `path`(propose/forward) + `action_code` + `schema_version=2`**（蒸馏 α/端侧小模型训练标的需要）。

### ⚠️ 阶段G 冲突检查

| # | 风险 | 对策 |
|---|---|---|
| 1 | α loop 长 LLM 调用阻塞 event loop → 心跳超时 → B 误判 A 断连 | `translate` 用 `asyncio.to_thread`；α loop 与心跳 ping **独立 asyncio Task** |
| 2 | α 输出 JSON 不合法 | schema 校验；失败抛 `TranslateError` → loop 下发 `hover` |
| 3 | α 输出含 `keep_clear_distance`/航点 | prompt 约束；schema 校验拒绝未知字段 |
| 4 | DeepSeek API key 缺失 | `.env` 未填 → 优雅降级为 hover + link_status 告警 |
| 5 | `metadata.approved` 未写 → 训练标的不可用 | `run_agent_with_log` 写 α 时必带 `approved/path/action_code` |
| 6 | α loop 异常疯狂重启 | catch + **退避重启(1~5s)** + 期间 B 维持悬停 |

**✅ S3 验收门**：A↔B ping/pong + action 下发 + pose 上行。
**✅ S5 前半**：hardcoded intent → α → ActionCommand → 假无人机。

## 阶段H — β Agent + SSE + 提议审核（完整 S5）

> **理由**：α 通了，现在接 β 中枢。β 是人类唯一对话对象。

### H.1 β Agent + 工具集（先最小集）

`agents/beta.py` + `tools/beta_tools.py`（历史查询 + 实时状态 + α 调度；analytics/dashboard 延后阶段L） + `prompts/beta.md`。

### H.2 SSE + 提议审核

`web/sse.py`(`POST /api/chat/beta` → SSE `text/tool_call_start/tool_call_result/plan(actions=动作意图概要,summary,proposalId)/error`) + `web/routes.py`（`/api/proposals/*/approve` — **C3 单一路径**，**废弃 `/api/plan/approve`**）。

### H.3 WS 下行

`web/ws.py`：下行 `pose/status/reject/link_status/alpha_output/alert/dashboard_config/voice_*/sync_response`；`sync` 响应一次性补齐 `current_pose+flight.*+current_action_plan`。

### H.4 β→α 两条路径（安全边界关键）

- `propose_to_alpha`：**总线层拦截** → 存 `pending_proposal` → 前端展示待审 → 人确认 → 才进 `alpha_input_queue`；
- `forward_last_human_message`：直接进 α 队列（免审，人已发话）。

### ⚠️ 阶段H 冲突检查

| # | 风险 | 对策 |
|---|---|---|
| 1 | propose 路径未经人审就进 α | **总线层拦截**：先存 `pending_proposal`，人审后才 append 队列 |
| 2 | `forward` 误要求人审 | 免审；metadata `path="forward", approved=true` |
| 3 | `plan` SSE 事件含航点段 | `actions` 为**动作意图概要**，非航点；`FlightPlanSegment` 已废弃 |
| 4 | `abort` 误走 α/LLM | `abort` 是**控制层操作,不走 LLM**；直接 `bus.call(to="small_model", tool="abort")` <100ms |
| 5 | `link_status.link` 枚举缺项 | 加 `'small_model'\|'ego_planner'` |

**✅ S5 验收门**（完整）：β Chat → α → 假无人机 + 系统消息。

## 阶段K — 安全兜底与 reject 回路（S4 + S7）

> **理由**：核心功能跑通后，专门验证"出错时无人机不乱飞"——航空安全核心。

### K.1 reject 回路（S4）

A 下发含**未知动作编码**的 ActionCommand → B stub 抛 `GoalGenError` → 上行 `event:reject` → A 注入 α 上下文 + WS 推前端。**碰撞校验回路延后到阶段M**。

### K.2 断连安全（S7）

- 杀 A → B 1s 内 `ipc_connected=False` → small_model 切 hover → 假无人机悬停；
- 杀 B → 假无人机 >500ms 无 setpoint → 自动悬停 → A 收 IPC 断连 → 前端 `link_status{state:"down"}`；
- 大模型 API 失败 → 不下发新意图（维持悬停）+ 告警。

### ⚠️ 阶段K 冲突检查

| # | 风险 | 对策 |
|---|---|---|
| 1 | 假无人机无超时悬停 → B 崩溃时乱飞 | **必须在阶段E 实现**（§9.1） |
| 2 | B 侧 msgpack 解帧失败导致断连 | 丢弃该帧，**不断连** |
| 3 | α loop 退避重启期 B 收不到意图 | B 维持悬停 |

**✅ S4 验收门**：reject 回路通。
